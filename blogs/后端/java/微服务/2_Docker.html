    # docker

    # docker基础

    # 命令解读

    docker run -d

    • run 创建容器
    • -d 后台运行
    • --name 起名
    • -p 端口映射 主机端口:容器端口
    • -e 环境变量

    镜像名称结构

    # 常见命令

    docker 拼接

    • pull 拉取镜像
    • images 查看镜像
    • rmi 删除镜像
    • build 打包镜像
    • save 保存镜像
    • load 加载镜像
    • push 推送镜像
    • run 创建容器
    • stop 停止容器
    • start 启动容器
    • ps 查看容器运行状态
    • rm 删除容器

    docker exec -it 容器 bash

    • exec 执行
    • -it 可输入的命令系统
    • bash 黑窗口

    # 命令别名

    vi ～/.bashrc

    可以自己设置 到处都有TypeOf

    # 数据卷挂载

    容器内目录与宿主机之间映射的桥梁

    只需要修改宿主机的内容就可以了

    docker volume

    • ls 查看数据卷
    • rm 删除数据卷
    • inspect 查看数据卷详情
    • prune 删除未使用的数据卷

    # 本地目录挂载

    本地查看容器是否有数据卷

    docker run

    • -v 本地目录:容器内目录
      • example: -v ./mysql:/var/lib/mysql

    任务：挂载

    通过挂载可以直接把数据库的表和数据放进去吗？

    # DockerFile 语法

    镜像就是包含了应用程序、程序运行的系统函数库、运行配置等文件的文件包。 构建镜像就是把上述文件打包的过程。

    部署一个java应用的步骤:

    1. 准备一个linux服务器
    2. 安装jre并配置环境变量
    3. 拷贝jar包
    4. 运行jar包

    构建一个java镜像的步骤

    1. 准备一个linux服务器
    2. 安装jre并配置环境变量
    3. 拷贝jar包
    4. 编写运行脚本

    镜像结构：

    入口：Entrypoint

    层：Layer 添加安装包、依赖、配置等、每次操作都形成新的一层。

    基础镜像：BaseImage

    # DockerFile指令

    • FROM 指定基础镜像
    • ENV 设置环境变量
    • COPY 拷贝本地文件到镜像的指定目录
    • RUN 执行linux的shell命令
    • EXPOSE 指定容器运行时监听的端口
    • ENTRYPOINT 镜像中应用的启动命令，容器运行时调用

    利用dockerFile描述镜像结构，记得备份就行了

    # 自定义镜像

    docker build

    • -t 镜像起名
    • . 指定Dockerfile所在目录

    好像是可以把自己的项目部署上去

    # 容器网络互联

    所有容器都是以bridge链接到Docker的一个虚拟网桥上

    docker0 网桥

    自定义网络

    通过ip addr获得信息

    通过自定义，可以很方便访问自己，相当于一个局域网

    docker network

    • create 创建一个网络
    • ls 查看所有网络
    • rm 删除指定网络
    • prune 清除未使用网络
    • connect 指定容器连接加入某网络
    • disconnect 指定容器连接离开某网络
    • inspect 查看网络详细信息

    # 项目部署

    # 部署java应用

    打包jar包，然后传到服务器，然后进行build

    # 部署前端

    nginx，创建容器挂载，同一个网络

    # DockerCompose

    手动部署太麻烦了

    引出docker compose，通过一个单独的yaml文件，来定义一组相关联的应用容器， 帮助我们实现多个相互关联的docker容器的快速部署

    项目，服务

    # 自己实践

    1. 如何将本地mysql的表数据同步服务器，使用datagrip，用mysqldump导出sql，使用mysql导入sql
    2. 本地项目打包成jar包以及dockerfile，传输到服务器，使用scp命令，私钥传输
    3. 在Linux服务器进行了项目部署，先build，后run，看启动日志，测试接口请求的正确性
    4. 部署了前端，使用nginx，自己试了试，最后发现还是之前自己配置的好呀

    学习｜SpringCloud｜Nacos配置管理

    Fang 2024/6/16 后端Java微服务

    # 配置管理

    # 什么是配置管理

    重复配置太多，没必要一个个的配置

    配置管理服务：业务配置/路由配置

    # 共享配置

    nacos：

    • 添加配置到配置列表
      • 变化的写在
      • 相应模块的资源文件里面，动态进行读取还能通过 ：执行默认的配置
    • 拉取共享配置
      • 基于NacosConfig拉取共享配置代替微服务的本地配置
      • 有一套SpringCloud上下文和SpringBoot上下文
      • 引入依赖
      • 新建bootstrap.yml

    # 配置热更新

    配置热更新：当修改配置文件中的配置时，微服务无需重启即可使配置生效。

    前提条件：

    • nacos中要有一个与微服务名有关的配置文件。
      • name - active - ymal
    • 微服务中以特定方式读取需要的配置文件

    配置热更新有点像boss平台的动态控制，哈哈，难道boss平台也是一个nacos？好奇

    # 动态路由（拓展）

    • 读取配置
    • 监听配置

    读取+监听，读了一下nacos源码，实际上维护的这个类已经自动注入了

    • 项目启动，先拉取一次配置，并且添加配置监听器
    • 监听到配置变更，需要去更新路由表
    • 第一次读取到配置，也需要更新到路由表
    • 监听到路由信息后，可以利用RouteDefinitionWriter来更新路由表，学习一个特殊容器
    • 解析配置信息+转换
    • 删除旧的路由表
    • 更新路由表+记录路由id，便于下一次更新时删除

    有点像回调函数

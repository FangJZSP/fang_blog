    # 初识nodeJs

    # 不同的浏览器使用不同的js解析引擎

    # js操作dom和bom

    调用web api

    # 运行环境

    代码运行所需的必要环境

    内置api是运行环境提供的特殊接口，只能在所属的运行环境中被调用

    node.js可以 用于后端开发

    # 什么是node

    node.js是一个v8引擎的js运行环境

    # 前后端js区别

    浏览器是js的前端运行环境

    node.js是js的后端运行环境，所以无法调用dom和bom等浏览器内置api

    # nodeJs能做什么

    有多种框架可以用，还能读写操作数据库

    # 学习路线

    # 浏览器中的js学习路径

    js语法 + 内置api + 第三方库

    # node.js的学习路径

    js语法 + node.js的内置api + 第三方api模块

    # node.js环境安装

    # 安装包

    已经安装了，mac可以用homebrew下载

    # 终端中的快捷键

    方向键上 能定位上一次的终端命令

    使用tab键 可以快速补全路径，mac可以使用终端插件完成这个功能

    esc可以快速清空命令

    win cls清空 mac clear清空

    # 什么是fs

    用来操作文件的api

    fs.readFile(path[,options],callback)

    fs.writeFile() 写入

    # 判断读取是否成功

    # fs-向指定的文件写入内容

    # fs-处理路径问题

    执行node命令时，会用所处的目录进行拼接，可能出现问题

    动态拼接目录会出现问题

    以./ 或 ../ 命名的路径

    换成绝对路径就可以，但是每个人的目录也不一样，所以这样的方式也不是最好的、

    然后nodeJs也提供了一个方法来解决

    # 路径拼接问题

    __dirname 为当前文件所处的目录

    # path-使用path模块处理路径

    用来处理路径模块

    path.join() 将多个路径拼接成一个路径字符串

    path.basename() 用来从路径的字符串中，将文件名解析出来，获取最后一个位置的东西

    ../会抵消一层路径

    path.extname() 获得路径文件的拓展名
